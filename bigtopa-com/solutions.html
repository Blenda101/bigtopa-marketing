The Fit principle in action.

BigTopa's entitlement system doesn't change the code — it changes what each organization gets access to. That's why a Greek chapter and a creator can share the same infrastructure without sharing any of the same experience.

01
Same code, different experience

Every organization runs on the same application. Entitlements determine which features are visible, which terminology is used, and which workflows are unlocked — without any code changes.

02
Domain plans are additive baselines

A domain plan sets a minimum floor — the features your niche needs, pre-configured. The Plan Builder lets you add above that floor. You can never remove what the domain requires.

03
Nomenclature is part of Fit

Greek chapters call events "Chapter Events." Reunion organizers call them "Gatherings." The underlying feature is identical — only the label changes. This is configured at the plan level, not the code level.

04
Revenue always flows to the hub

Regardless of how a plan is composed, all financial activity from rooms and spaces aggregates to the Tenant Admin Hub — a single financial summary across every space your organization runs.
